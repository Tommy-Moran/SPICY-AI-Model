# Code Availability Statement

## Repository

All analysis code underpinning this study is published in a dedicated GitHub repository:

`https://github.com/Tommy-Moran/SPICY-AI-Model`

The repository includes:

- `analysis_v3.py` — fully reproducible statistical pipeline (pandas / scipy / matplotlib). Reads the de-identified Likert score data from `data/scores_long_v3.csv` and recomputes every Mann-Whitney U statistic (with Holm-Bonferroni adjustment across the 12 score comparisons), every Fisher exact p-value, and every published figure. Running `python3 analysis_v3.py` from a clean clone reproduces Tables 2 and all four figures exactly as published.
- `data/scores_long_v3.csv` — 150-row de-identified Likert score dataset (50 cases × 3 plan sources × 6 scoring variables).
- `data/demographics.csv` — 50-row de-identified patient demographics (RIN, age, sex, cardiovascular risk factors, ACR; lipid/HbA1c/BMI/SBP fields are sparsely populated where the original RACPC record did not capture them).
- `data/summary_v3.csv`, `data/mannwhitney_v3.csv`, `data/error_rates_v3.csv`, `data/error_summary_v3.csv`, `data/stats_output_v3.txt` — machine-readable statistical outputs regenerated by the analysis script.
- `figures/` — 300-DPI PNG, PDF, and TIFF artefacts of all four figures.
- `SPICY_AI/prompt.txt` — full verbatim system prompt used to instruct SPICY-AI.
- `SPICY_AI/spicy_ai.py` — RAG-LLM inference pipeline (LangChain / FAISS / Ollama) with pinned model digest, embedding revision, and retrieval seed.
- `SPICY_AI/rag_config.py` — knowledge-base manifest and retrieval configuration.
- `SPICY_AI/knowledge_base/download_guidelines.sh` — downloads the publicly available guideline PDFs that form the guideline-only KB.
- `SPICY_AI/Dockerfile` — reproducibility container.
- `requirements.txt` and `SPICY_AI/requirements_spicy.txt` — version-pinned Python dependency manifests.

The analysis pipeline runs end-to-end on Python 3.10+ (`pip install -r requirements.txt`) using only the de-identified CSVs included in the repo. An optional `--source-xlsx` flag rebuilds the long-format CSV from the raw scoring workbooks (held by the authors and not redistributed; see Data Sharing Statement).

## SPICY-AI model

SPICY-AI is a locally deployed RAG-LLM purpose-built for the Royal Hobart Hospital RACPC workflow. The system integrates:

1. A curated knowledge base comprising the 2021 American Heart Association Chest Pain Guidelines, the 2021 AHA/ACC Coronary Artery Disease Guidelines, the 2021 CDC Dyslipidaemia Guidelines, and the RHH RACPC institutional protocols (four documents total).
2. A retrieval layer (sentence-transformer embeddings + FAISS vector store) returning grounding passages at inference time. Embedding model and revision, chunk size, overlap, and `top_k` are pinned in `SPICY_AI/spicy_ai.py`.
3. A base LLM hosted on institutional hardware. The exact tag used in the published study is `deepseek-r1:14b` (Q4_K_M quantisation via Ollama); the model digest is recorded in `SPICY_AI/spicy_ai.py` so reproducers can verify (`ollama show deepseek-r1:14b --modelfile`) before running.

Because the knowledge base contains institution-specific protocols and because the system is designed to run inside the RHH network, the full SPICY-AI stack is not open-sourced. However, the authors share:

- The published-guideline-only portion of the knowledge base (downloadable via `SPICY_AI/knowledge_base/download_guidelines.sh`).
- The verbatim prompt template and pinned retrieval configuration.
- A Dockerfile and instructions for reproducing the retrieval and generation pipeline against a publicly available base model.

Requests for the institutional protocol can be directed to the corresponding author and will be accommodated subject to RHH data-governance approval.

## Third-party models

GPT-4o was accessed via the OpenAI API at the time of the study (September–October 2024) and is not redistributed here. The prompt template used for GPT-4o is included in the GitHub repository.
